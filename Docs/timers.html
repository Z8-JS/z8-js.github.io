Timers

Z8 provides global timer functions to schedule code execution after a set period of time. These are managed by Z8's high-performance internal event loop.

setTimeout(callback, delay[, ...args])

Schedules execution of a one-time callback after delay milliseconds.

  • callback: The function to run.
  • delay: Number of milliseconds to wait.
  • ...args: Optional arguments to pass to the callback.

Returns a numeric timeoutId.

setTimeout((name) => {
    console.log("Hello", name);
}, 1000, "Zane");

clearTimeout(timeoutId)

Cancels a timeout previously established by calling setTimeout().

const id = setTimeout(cb, 1000);
clearTimeout(id);

Z8's event loop is designed with a "Sleep-to-Efficiency" model. When no timers are immediately due, the process sleeps at the OS level to minimize CPU usage, waking up instantly when the next timer expires or a new one is added.

In massive timer stress tests, Z8's event loop shows lower management overhead compared to complex runtimes, allowing for tighter scheduling and better predictability in CLI tools.
